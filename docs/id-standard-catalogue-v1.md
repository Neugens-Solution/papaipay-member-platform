# PAPAIPAY Portal V1 ID Standard Catalogue

This catalogue defines public reference ID formats for searchable, copyable records. Internal database primary keys should use UUIDs. Public references should be generated by backend services and stored with unique indexes.

## ID principles

- Keep UUID as internal primary key.
- Store public reference as a separate unique field.
- Never infer database identity from public sequence numbers.
- Public references must be visible in Admin tables, detail pages, reports, and exports.
- Public references should be copyable from detail pages.
- Use uppercase prefixes and zero-padded numeric suffixes.
- Campaign Code is a business code and should remain separate from Campaign ID.

## Generation rules

1. Generate UUID first.
2. Generate public reference in the same transaction.
3. Enforce unique index on public reference.
4. Never reuse deleted public references.
5. Keep public reference immutable after creation.
6. Campaign Code may be editable only before publish, subject to audit log.
